# Security Policy

## ⚠️ Intentionally Vulnerable Code

**This repository contains intentionally vulnerable code for educational purposes.**

## Reporting Security Issues

If you discover a security vulnerability in the **infrastructure** of this repository (not the intentional vulnerabilities), please report it to:

- **Email**: appsec@veritran.com
- **Subject**: [SECURITY] vtrust-skills-vulnerable-labs

Please do NOT open public issues for actual security vulnerabilities in the repository infrastructure.

## Scope

### ✅ IN SCOPE (Report These)
- Security issues in the repository infrastructure
- Actual vulnerabilities in documentation or build scripts
- Security misconfigurations in CI/CD

### ❌ OUT OF SCOPE (Do Not Report)
- Intentional vulnerabilities in `/skills/*/vulnerable/` directories
- Known vulnerable patterns (that's the point!)
- Missing security controls in example code

## Usage Restrictions

This code is for:
- ✅ Security training
- ✅ Educational purposes
- ✅ Testing security tools
- ✅ Vulnerability research

This code is NOT for:
- ❌ Production use
- ❌ Processing real user data
- ❌ Malicious purposes
- ❌ Unauthorized testing

## Legal Disclaimer

Users of this repository agree to:
1. Use code only in controlled environments
2. Not deploy vulnerable skills in production
3. Not use for unauthorized access
4. Comply with applicable laws and regulations

## Responsible Disclosure

If you discover a novel vulnerability pattern not covered in this repository, please:
1. Document the vulnerability
2. Create a proof of concept
3. Submit a pull request with both vulnerable and secure versions
4. Follow our [Contributing Guidelines](CONTRIBUTING.md)

**Remember**: With great power comes great responsibility. Use these skills ethically.
